🔎 관리자 사전 점검 가이드 — 미노출

개발자 에스컬레이션 전, 관리자(기획·운영자)가 직접 확인할 수 있는 5단계 점검입니다. 각 단계를 순서대로 진행하고 이상이 없으면 다음 단계로 넘어가세요.

PHASE 1 GFD 어드민 — 기본 상태 점검
확인 항목 점검 방법 & 기대값
캠페인 상태GFD 어드민 > 캠페인 목록 > 상태 칼럼 = 운영중 확인 (일시중지·종료 아닌지)
광고그룹 상태해당 광고그룹 상태 = 운영중 확인 (예산 소진으로 자동 중지 여부 확인)
소재 상태소재 심사 상태 = 승인 (심사중·반려 시 미노출 정상)
일 예산 / 전체 예산잔여 예산 > 0 확인. 당일 소진 시 익일 자정 리셋됨
캠페인 기간시작일 ≤ 오늘 ≤ 종료일 확인. 종료일 경과 시 자동 종료
광고그룹 노출 시간대현재 시각이 설정된 노출 허용 시간대 내인지 확인

✅ PHASE 1 통과 기준: 위 항목 전체 이상 없음 → PHASE 2로 이동
❌ 이상 발견 시: 해당 항목 수정 후 재확인. 개발자 에스컬레이션 불필요.

PHASE 2 소재 적합성 점검
확인 항목 점검 방법 & 기대값
소재 사이즈 vs 지면 사이즈GFD 어드민 > 소재 상세 > 가로×세로 px 확인. 게재 지면 규격과 일치 여부 대조 (예: 300×250, 320×50)
소재 유형 vs 지면 지원 유형배너/네이티브/동영상 유형이 해당 지면에서 지원하는 유형인지 GFP 어드민 지면 설정과 대조
VAST URL 직접 호출 (동영상)브라우저에서 VAST URL 직접 입력 → 200 응답 & XML 반환 확인. 오류 시 소재 자체 문제
이미지/HTML 소재 URL 접근성소재 이미지 URL 직접 접근 → 200 응답 확인. CDN 만료·도메인 오류 시 소재 교체 필요

✅ PHASE 2 통과 기준: 사이즈 일치, 유형 일치, URL 200 응답 → PHASE 3으로 이동
❌ 이상 발견 시: 소재 수정·재업로드 또는 지면 변경. 개발자 에스컬레이션 불필요.

PHASE 3 타겟팅 & 경쟁 환경 점검
확인 항목 점검 방법 & 기대값
타겟팅 조건 범위성별·연령·지역·디바이스 타겟팅이 지나치게 좁지 않은지 확인. 도·시 단위보다 넓게 설정 권장
CPM 입찰가 경쟁력동일 지면의 타 광고 CPM 대비 입찰가가 현저히 낮은 경우 낙찰 불가. CPM ↑ 또는 CPTA/CPTR 확인
Frequency Cap 설정Frequency Cap이 너무 낮게 설정되어 타겟 모수가 소진된 경우 노출 없음. Cap 값 상향 또는 제거 검토
Target Frequency와 혼동 주의Target Frequency(목표 도달 빈도)는 Frequency Cap(최대 노출 제한)과 다름. 설정 항목 재확인
지역 타겟팅 단위특정 구·동 단위 타겟팅 시 모수 극소화 가능. 시·도 단위 이상으로 확대 테스트 권장

✅ PHASE 3 통과 기준: 타겟 범위 적정, 입찰가 경쟁력 있음, Frequency 설정 정상 → PHASE 4로 이동
❌ 이상 발견 시: 타겟 조정 또는 입찰가 수정. 개발자 에스컬레이션 불필요.

PHASE 4 GFP 어드민 — 미디에이션 설정 점검
확인 항목 점검 방법 & 기대값
NAVER_DIRECT 활성화 여부GFP 어드민 > 미디에이션 설정 > NAVER_DIRECT 항목 = 활성화 확인
지면 배너/네이티브 설정GFP 지면에서 배너 또는 네이티브가 허용 유형으로 등록되어 있는지 확인
GFP 지면 사이즈 등록GFP 어드민 > 지면 상세 > 허용 사이즈 목록에 소재 사이즈가 포함되어 있는지 확인
GFP SDK 버전퍼블리셔 앱/페이지에서 사용 중인 GFP SDK 버전이 최신 안정 버전인지 확인 (퍼블리셔 문의 필요할 수 있음)

✅ PHASE 4 통과 기준: NAVER_DIRECT 활성, 유형·사이즈 등록 정상 → PHASE 5로 이동
❌ 이상 발견 시: GFP 어드민에서 직접 수정 가능. 설정 변경 후 재확인.

PHASE 5 환경 확인 & 최종 판정
확인 항목 점검 방법 & 기대값
Real / Beta / Dev 환경 구분테스트 환경(Beta/Dev)에서만 미노출인지, Real 환경에서도 동일한지 확인. 환경별 설정이 다를 수 있음
리포트 집계 시간 지연GFD 리포트는 최대 1~2시간 지연될 수 있음. 실시간 노출 여부는 GFP 실시간 리포트 또는 Kibana로만 확인 가능
광고 노출 직접 확인실제 지면(앱/웹)에서 해당 광고가 노출되는지 직접 확인. 캐시 초기화 후 재시도 권장

🔎 관리자 사전 점검 가이드 — 과노출

개발자 에스컬레이션 전, 관리자(기획·운영자)가 직접 확인할 수 있는 4단계 점검입니다. 과송출이 심각할 경우 PHASE 1(긴급 대응)을 먼저 수행하세요.

PHASE 1 긴급 대응 — 과송출 즉시 판단
⚠️ 과송출 심각 기준: 일 예산 대비 실제 집행액이 120% 초과 또는 전일 동시간 대비 노출량 2배 이상
확인 항목 점검 방법 & 조치
즉시 일시중지 판단과송출 심각 시 GFD 어드민에서 광고그룹 즉시 일시중지. 광고주 고지 후 원인 파악 병행
배분 설정 확인광고그룹 > 배분 방식 = 균등 배분 권장. '빠른 배분' 설정 시 조기 소진 발생 가능
예측치 생성 시점캠페인 시작 직후 ~24시간은 예측치 미생성 구간. 이 기간 과집계처럼 보이는 건 정상 동작

✅ PHASE 1 완료 기준: 즉시 위협 없음 or 일시중지 조치 완료 → PHASE 2로 이동

PHASE 2 Frequency Cap 설정 점검
확인 항목 점검 방법 & 기대값
Frequency Cap 설정 여부GFD 어드민 > 광고그룹 > Frequency Cap 항목 설정 여부 확인. 미설정 시 동일 사용자 무제한 노출 가능
Cap 값 적정성일반적으로 일 3~5회, 주 10회 이하 권장. 브랜딩 목적의 경우도 일 10회 초과는 과노출 위험
Target Frequency 혼동 주의Target Frequency(도달 목표)는 Frequency Cap(상한 제한)이 아님. 두 설정 모두 점검

✅ PHASE 2 통과 기준: Frequency Cap 설정됨 & 적정 수준 → PHASE 3으로 이동
❌ 미설정 or 과도하게 높음 → Cap 즉시 설정/수정. 개발자 에스컬레이션 불필요.

PHASE 3 리포트 이상치 점검
확인 항목 점검 방법 & 기대값
전일/전주 동시간 노출량 비교GFD 리포트에서 동일 광고그룹 전일·전주 동시간대 노출 수치 추출 후 비교. 50% 이상 급증 시 이상 신호
DSP 리포트 대조GFD 리포트 수치와 DSP(광고주측) 리포트 수치 비교. 양측 모두 과다 → 실제 과노출. GFD만 과다 → 집계 오류 의심
이상 발생 시간대 특정시간별 리포트로 이상 발생 시간대 특정. 특정 시간대에만 집중된다면 스케줄 설정 오류 가능성

✅ PHASE 3 통과 기준: 이상 시간대 특정 완료 or 정상 범위 확인 → PHASE 4로 이동

PHASE 4 어드민 UI 중복 항목 점검 & 최종 판정
확인 항목 점검 방법 & 기대값
동일 소재 중복 등록 여부GFD 어드민에서 동일 광고그룹 내 중복 소재 목록 확인. 동일 소재가 여러 번 등록된 경우 노출 중복 집계 가능
중복 광고그룹 여부동일 캠페인 내 타겟·지면 설정이 완전히 동일한 복수 광고그룹 존재 시 노출 중복 발생 가능
GFP 미디에이션 중복 요청GFP 어드민에서 동일 지면에 NAVER_DIRECT가 여러 미디에이션 레이어에 등록된 경우 중복 요청 발생 가능

🏁 최종 판정 기준

✅ 관리자 자체 해결 가능

  • Frequency Cap 미설정 또는 과도한 값
  • 배분 방식 오설정 (빠른 배분)
  • 중복 소재·광고그룹 등록
  • 캠페인 시작 후 24시간 이내 (정상)

🔺 개발자 에스컬레이션 필요

  • PHASE 1~4 모두 이상 없음에도 과노출 지속
  • GFD 리포트와 DSP 리포트 수치 불일치
  • 특정 지면·시간대에서만 재현되는 급증

에스컬레이션 시 PHASE 1~4 점검 결과, 이상 발생 시간대, DSP 리포트 수치를 함께 전달하면 개발자 대응 속도가 크게 향상됩니다.
